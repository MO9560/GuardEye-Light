### 问题1：模型加载方式错误

**当前代码（Detector.kt 第69-74行）**
```kotlin
// ❌ 从文件系统加载，assets里的文件找不到
private fun loadModelFile(path: String): MappedByteBuffer {
    val file = java.io.File(path)  // path = "yolov8n.tflite" → 找不到
    ...
}
```

**参考方案A：直接用 TF Lite Task Library（推荐）**

参考官方示例 `ObjectDetectorHelper.kt`：
```kotlin
// ✅ 无需手动加载模型文件，Task Library自动从assets读取
val options = ObjectDetectorOptions.builder()
    .setBaseOptions(BaseOptions.builder().setNumThreads(4).build())
    .setScoreThreshold(0.45f)
    .setMaxResults(10)
    .build()

objectDetector = ObjectDetector.createFromFileAndOptions(
    context,
    "yolov8n.tflite",   // assets下的文件名
    options
)
```

**参考方案B：手动从 assets 加载**
```kotlin
// ✅ 正确方式：通过AssetFileDescriptor加载
private fun loadModelFile(context: Context, modelPath: String): MappedByteBuffer {
    val fileDescriptor = context.assets.openFd(modelPath)
    val inputStream = FileInputStream(fileDescriptor.fileDescriptor)
    val fileChannel = inputStream.channel
    val startOffset = fileDescriptor.startOffset
    val declaredLength = fileDescriptor.declaredLength
    return fileChannel.map(FileChannel.MapMode.READ_ONLY, startOffset, declaredLength)
}
```

### 问题2：输入归一化完全错误

**当前代码（Detector.kt 第100-104行）**
```kotlin
// ❌ ByteBuffer + 错误计算：(pixel shr 16 and 0xFF) / 255f * 255 = 原值
// ❌ 且yolov8n.tflite是FLOAT32模型，应该用FloatBuffer，不是ByteBuffer
imgData.put(((pixel shr 16 and 0xFF) / 255f * 255).toInt().toByte())
imgData.put(((pixel shr 8  and 0xFF) / 255f * 255).toInt().toByte())
imgData.put(((pixel and 0xFF) / 255f * 255).toInt().toByte())
```

**参考方案：用 Task Library（自动处理预处理）**

```kotlin
// ✅ 官方示例做法：TensorImage自动完成预处理
fun detect(bitmap: Bitmap): List<Detection>? {
    val tensorImage = TensorImage.fromBitmap(bitmap)
    // Task Library自动完成：resize + 归一化(0-255→0-1) + 维度排列
    val results = objectDetector?.detect(tensorImage) ?: return null
    // results 已经是结构化对象，无需手动解析输出张量
    return results.map { 
        Detection(
            label = it.categories[0].label,
            confidence = it.categories[0].score,
            box = floatArrayOf(it.boundingBox.left / bitmap.width, ...),
            isPolice = false // 颜色分析单独做
        )
    }
}
```

**如果坚持手动处理（不推荐）：**
```kotlin
// ✅ 正确手动预处理：FLOAT32 + 归一化0-1
val imgData = FloatBuffer.allocate(1 * 3 * INPUT_SIZE * INPUT_SIZE)
imgData.order(ByteOrder.nativeOrder())

val pixels = IntArray(INPUT_SIZE * INPUT_SIZE)
resized.getPixels(pixels, 0, INPUT_SIZE, 0, 0, INPUT_SIZE, INPUT_SIZE)
for (pixel in pixels) {
    imgData.put(((pixel shr 16) and 0xFF) / 255f)  // R / 255f
    imgData.put(((pixel shr 8)  and 0xFF) / 255f)  // G / 255f
    imgData.put((pixel and 0xFF) / 255f)             // B / 255f
}
```

### 问题3：输出解码错误

**当前代码（Detector.kt 第106-107行 + 第139-142行）**
```kotlin
// ❌ 输出维度错误：YOLOv8输出是 [1, 84, 8400]，不是[1, 85, 8400]
val output = Array(1) { Array(85) { FloatArray(8400) } }

// ❌ 归一化重复除以INPUT_SIZE：YOLO输出cx,cy,w,h已经是0-1
val cx = output[0][0][i] / INPUT_SIZE  // 不应该再除
```

**参考方案：用 Task Library（无需手动解析输出）**

Task Library 直接返回 `List<Detection>`，每个 Detection 包含：
- `boundingBox: RectF` — 像素坐标
- `categories: List<Category>` — 类别+置信度

**如果手动解析YOLOv8输出（不推荐）：**
```kotlin
// ✅ 正确输出维度：[1, 84, 8400]
// 84 = 4(cx,cy,w,h) + 80(COCO类别)  ← 注意是84不是85
// 或 85 = 4 + 1(置信度objConf) + 80类别  ← 看模型训练方式
val output = Array(1) { Array(84) { FloatArray(8400) } }
interp.run(imgData, output)

// ✅ cx,cy,w,h已经是0-1，不要再除INPUT_SIZE
val cx = output[0][0][i]
val cy = output[0][1][i]
val w  = output[0][2][i]
val h  = output[0][3][i]
```

### 问题4：Telegram 发送实现错误

**当前代码（BotManager.kt）问题：**
- MIME类型拼写错误 `image/jpeg`（应为 `image/jpeg`）
- 使用 `java.net.HttpURLConnection` 手动拼装 multipart，容易出错
- 无重试机制

**参考方案：用 OkHttp（Retired-Sentinel-App 的做法）**

```kotlin
// ✅ Retired-Sentinel-App 使用的 OkHttp 方式
private val client = OkHttpClient()

fun sendPhoto(bitmap: Bitmap, caption: String) {
    val bos = ByteArrayOutputStream()
    bitmap.compress(Bitmap.CompressFormat.JPEG, 90, bos)
    val imageBytes = bos.toByteArray()
    
    val requestBody = MultipartBody.Builder()
        .setType(MultipartBody.FORM)
        .addFormDataPart("chat_id", CHAT_ID)
        .addFormDataPart("caption", caption)
        .addFormDataPart(
            "photo",
            "guard_${System.currentTimeMillis()}.jpg",
            RequestBody.create(MediaType.parse("image/jpeg"), imageBytes)
        )
        .build()
    
    val request = Request.Builder()
        .url("https://api.telegram.org/bot${BOT_TOKEN}/sendPhoto")
        .post(requestBody)
        .build()
    
    client.newCall(request).enqueue(object : Callback {
        override fun onFailure(call: Call, e: IOException) {
            Log.e("BotManager", "发送失败", e)
        }
        override fun onResponse(call: Call, response: Response) {
            Log.d("BotManager", "发送成功: ${response.body()?.string()}")
        }
    })
}
```

依赖：`implementation 'com.squareup.okhttp3:okhttp:4.12.0'`

### 问题5：后台保活机制

**参考方案：Foreground Service + WakeLock（Retired-Sentinel-App 做法）**

```kotlin
// ✅ 正确保活方式
class MonitorForegroundService : Service() {
    
    private val NOTIFICATION_ID = 10001
    private val CHANNEL_ID = "guardeye_monitor"
    
    override fun onCreate() {
        super.onCreate()
        createNotificationChannel()
        startForeground(NOTIFICATION_ID, buildNotification("GuardEye 监控中"))
        
        // 保持CPU不休眠
        val wakeLock = (getSystemService(Context.POWER_SERVICE) as PowerManager)
            .newWakeLock(PowerManager.PARTIAL_WAKE_LOCK, "GuardEye:Monitor")
        wakeLock.acquire(10*60*1000L /*10分钟*/)
    }
    
    private fun createNotificationChannel() {
        val channel = NotificationChannel(
            CHANNEL_ID, "GuardEye监控", NotificationManager.IMPORTANCE_LOW
        )
        getSystemService(NotificationManager::class.java).createNotificationChannel(channel)
    }
}
```

**AndroidManifest.xml 需要添加的权限：**
```xml
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_CAMERA" />
<uses-permission android:name="android.permission.WAKE_LOCK" />
<uses-permission android:name="android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS" />
```

**优点：** 对模型输出有更多控制（如自定义NMS）  
**改动范围：** 修复 `Detector.kt` 的3个Bug

```
修复点：
1. loadModelFile() → 改用 AssetFileDescriptor
2. ByteBuffer → FloatBuffer，正确归一化0-1
3. 输出维度 [1,84,8400] 或 [1,85,8400]（需确认模型）
4. 去掉 cx,cy,w,h 的 / INPUT_SIZE
```

**优点：** 经过实际验证的完整方案
